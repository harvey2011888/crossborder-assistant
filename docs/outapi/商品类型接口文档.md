# 商品类型接口文档

## 1. 接口概述

快递类型接口用于获取系统支持的快递类型列表，包括国内快递和国际快递。该接口返回所有可用的快递类型信息，为用户提供选择快递类型的参考。

## 2. 接口路径

```
POST https://express-api-727446398583.us-central1.run.app/express/pub/types
```

## 3. 请求头

| 头部名称            | 类型       | 必填 | 描述             |
| :-------------- | :------- | :- | :------------- |
| `nonce`         | `string` | 否  | 随机字符串，用于防止重放攻击 |
| `Authorization` | `string` | 否  | 认证令牌（如果需要）     |

## 4. 请求参数

该接口为 GET 请求，无请求体参数。

## 5. 响应参数

### 5.1 基本参数

| 参数名                | 类型       | 描述                  |
| :----------------- | :------- | :------------------ |
| `code`             | `number` | 响应状态码               |
| `msg`              | `string` | 响应消息                |
| `data`             | `array`  | 快递类型列表              |
| `data[].id`        | `number` | 快递类型ID              |
| `data[].name`      | `string` | 快递类型名称              |
| `data[].code`      | `string` | 快递类型代码              |
| `data[].type`      | `number` | 快递类型（1-国内快递，2-国际快递） |
| `data[].status`    | `number` | 状态（1-启用，0-禁用）       |
| `data[].sort`      | `number` | 排序权重                |
| `data[].createdAt` | `string` | 创建时间                |
| `data[].updatedAt` | `string` | 更新时间                |

## 6. 业务流程

1. 接收请求，验证请求头中的 nonce 参数
2. 从数据库或缓存中获取快递类型列表
3. 对快递类型列表进行排序和筛选
4. 返回快递类型列表

## 7. 错误码

| 错误码 | 错误信息                 | 描述          |
| :-- | :------------------- | :---------- |
| 411 | missing nonce header | 缺少 nonce 头部 |
| 400 | 参数错误                 | 请求参数不合法     |
| 401 | 未授权                  | 用户未登录或登录已过期 |
| 500 | 系统错误                 | 服务器内部错误     |

## 8. 接口调用示例

### 8.1 cURL示例

```bash
curl -X GET "https://api.hubbuy.app/hubbuy_express/express/pub/types" \
  -H "nonce: random_string_here" \
  -H "Authorization: Bearer your_token_here"
```

### 8.2 JavaScript示例

```javascript
fetch('https://api.hubbuy.app/hubbuy_express/express/pub/types', {
  method: 'GET',
  headers: {
    'nonce': 'random_string_here',
    'Authorization': 'Bearer your_token_here'
  }
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error('Error:', error));
```

## 9. 注意事项

1. 必须提供 nonce 头部，否则会返回 411 错误
2. nonce 值应该是随机生成的，每次请求都应该不同，以防止重放攻击
3. 接口返回的快递类型列表可能会根据系统配置而变化
4. 前端在使用快递类型时，应该根据 type 字段区分国内快递和国际快递
5. 应该只显示 status 为 1（启用）的快递类型
