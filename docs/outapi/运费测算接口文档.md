# 运费测算接口文档

## 1. 接口概述

运费测算接口用于计算订单的运费，包括国内运费和国际运费。该接口支持根据商品重量、体积、目的地等因素计算运费，为用户提供准确的运费估算。

## 2. 接口路径

```
POST https://express-api-727446398583.us-central1.run.app/express/pub/postage
```

## 3. 请求头

| 头部名称            | 类型       | 必填 | 描述                            |
| :-------------- | :------- | :- | :---------------------------- |
| `Content-Type`  | `string` | 是  | 请求内容类型，固定为 `application/json` |
| `nonce`         | `string` | 否  | 随机字符串，用于防止重放攻击                |
| `Authorization` | `string` | 否  | 认证令牌（如果需要）                    |

## 4. 请求参数

### 4.1 基本参数

| 参数名                    | 类型        | 必填 | 描述        |
| :--------------------- | :-------- | :- | :-------- |
| `goodsList`            | `array`   | 是  | 商品列表      |
| `goodsList[].id`       | `string`  | 是  | 商品ID      |
| `goodsList[].weight`   | `number`  | 是  | 商品重量（克）   |
| `goodsList[].length`   | `number`  | 是  | 商品长度（厘米）  |
| `goodsList[].width`    | `number`  | 是  | 商品宽度（厘米）  |
| `goodsList[].height`   | `number`  | 是  | 商品高度（厘米）  |
| `goodsList[].quantity` | `number`  | 是  | 商品数量      |
| `fromCountry`          | `string`  | 是  | 发货国家/地区代码 |
| `fromProvince`         | `string`  | 是  | 发货省份      |
| `fromCity`             | `string`  | 是  | 发货城市      |
| `toCountry`            | `string`  | 是  | 收货国家/地区代码 |
| `toProvince`           | `string`  | 是  | 收货省份      |
| `toCity`               | `string`  | 是  | 收货城市      |
| `expressType`          | `number`  | 是  | 快递类型ID    |
| `isInsurance`          | `boolean` | 否  | 是否购买保险    |
| `insuranceAmount`      | `number`  | 否  | 保险金额      |

### 4.2 请求示例

```json
{"country":"US","categoryTypes":[189],"requestType":1,"arr":[{"weight":100,"long":10,"width":10,"height":10,"count":1,"coefficient":1}]}
```

## 5. 响应参数

### 5.1 基本参数

| 参数名                          | 类型       | 描述      |
| :--------------------------- | :------- | :------ |
| `code`                       | `number` | 响应状态码   |
| `msg`                        | `string` | 响应消息    |
| `data`                       | `object` | 响应数据    |
| `data.deliveryAmt`           | `string` | 国内运费（元） |
| `data.internationalShipping` | `string` | 国际运费（元） |
| `data.totalAmt`              | `string` | 总运费（元）  |
| `data.insuranceFee`          | `string` | 保险费用（元） |
| `data.estimatedDays`         | `number` | 预计送达天数  |
| `data.expressInfo`           | `object` | 快递公司信息  |
| `data.expressInfo.id`        | `number` | 快递公司ID  |
| `data.expressInfo.name`      | `string` | 快递公司名称  |
| `data.expressInfo.code`      | `string` | 快递公司代码  |

## 6. 业务流程

1. 接收运费测算请求，验证请求头中的 nonce 参数
2. 验证请求参数的合法性
3. 根据商品信息计算总重量和总体积
4. 根据发货地和收货地确定运输路线
5. 根据快递类型和重量/体积计算基础运费
6. 计算国际运费（如果适用）
7. 计算保险费用（如果购买）
8. 计算总运费和预计送达时间
9. 返回测算结果

## 7. 错误码

| 错误码 | 错误信息                 | 描述          |
| :-- | :------------------- | :---------- |
| 411 | missing nonce header | 缺少 nonce 头部 |
| 400 | 参数错误                 | 请求参数不合法     |
| 401 | 未授权                  | 用户未登录或登录已过期 |
| 500 | 系统错误                 | 服务器内部错误     |
| 501 | 快递公司未配置              | 选择的快递类型未配置  |
| 502 | 运费计算失败               | 运费计算过程中出现错误 |

## 9. 注意事项

1. 运费测算结果仅供参考，实际运费可能会因实际重量、体积、燃油附加费等因素有所不同
2. 国际运费可能会受到目的地国家/地区的关税、税费等因素影响
3. 保险费用通常为保险金额的一定比例，具体比例根据快递公司而定
4. 预计送达时间为参考值，实际送达时间可能会因天气、海关清关等因素有所延迟
5. 对于特殊商品（如易燃易爆物品、危险品等），可能需要特殊处理，运费计算方式也会有所不同
6. 必须提供 nonce 头部，否则会返回 411 错误
7. nonce 值应该是随机生成的，每次请求都应该不同，以防止重放攻击
